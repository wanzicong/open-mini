# Open Mini 使用指南

## 一、项目简介

**Open Mini** 是一个极简版的 AI 编程助手，完全支持 MiniMax Code Plan CN 模型。

### 核心特性

- 🤖 单一模型专注 - 只支持 MiniMax Code Plan CN
- 💻 跨平台 - Windows、Linux、macOS
- 🔧 6 个内置工具 - Read、Write、Edit、Bash、Glob、Grep
- 💬 交互式对话 - 支持命令行持续对话
- ⚡ 轻量级 - 依赖少，启动快

---

## 二、环境准备

### 1. 安装 Bun

Open Mini 使用 Bun 作为运行时。

```bash
# macOS/Linux
curl -fsSL https://bun.sh/install | bash

# Windows (PowerShell)
irm bun.sh | iex
```

### 2. 克隆项目

```bash
git clone https://github.com/wanzicong/open-mini.git
cd open-mini
```

### 3. 安装依赖

```bash
bun install
```

### 4. 配置 API Key

在项目根目录创建 `.env` 文件：

```
MINIMAX_API_KEY=你的API密钥
```

> 📝 **注意**：`.env` 文件已添加到 `.gitignore`，不会上传到 GitHub

## 三、快速开始

### 方式一：单次任务

```bash
# 计算
bun run src/cli/index.ts "计算 123 + 456"

# 回答问题
bun run src/cli/index.ts "用一句话介绍 TypeScript"

# 创建文件
bun run src/cli/index.ts "在当前目录创建 hello.txt，内容为 Hello World"

# 读取文件
bun run src/cli/index.ts "读取 package.json 文件"

# 搜索文件
bun run src/cli/index.ts "搜索当前目录下所有的 .ts 文件"
```

### 方式二：交互式对话

```bash
bun run src/cli/index.ts
# 或
bun run src/cli/index.ts -i
```

交互式命令：
- `exit` / `quit` - 退出程序
- `clear` - 清除对话历史

## 四、命令行参数

| 参数 | 简写 | 说明 | 示例 |
|------|------|------|------|
| `--help` | `-h` | 显示帮助 | `bun run src/cli/index.ts --help` |
| `--version` | `-v` | 显示版本 | `bun run src/cli/index.ts -v` |
| `--interactive` | `-i` | 交互式模式 | `bun run src/cli/index.ts -i` |
| `--project` | - | 指定项目目录 | `bun run src/cli/index.ts --project /path/to/project "任务"` |
| `--model` | - | 指定模型 | `bun run src/cli/index.ts --model MiniMax-M2.5-highspeed "任务"` |
| `--max-iterations` | - | 最大迭代次数 | `bun run src/cli/index.ts --max-iterations 10 "任务"` |

## 五、配置文件

### .open-mini.json

在项目根目录创建 `.open-mini.json` 配置文件：

```json
{
  "model": "MiniMax-M2.5-highspeed",
  "temperature": 0.7,
  "maxTokens": 8192,
  "baseURL": "https://api.minimaxi.com/anthropic/v1"
}
```

### 可用配置项

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `model` | string | MiniMax-M2.5-highspeed | 模型名称 |
| `temperature` | number | 0.7 | 温度参数 |
| `maxTokens` | number | 8192 | 最大输出 tokens |
| `baseURL` | string | API 地址 | MiniMax API 地址 |

## 七、工具详解

### 1. read - 读取文件

读取指定文件的内容。

```
参数: filePath (文件路径)
示例: 读取 package.json
```

### 2. write - 写入文件

创建新文件或覆盖已有文件。

```
参数: filePath (文件路径), content (文件内容)
示例: 创建 config.json
```

### 3. edit - 编辑文件

精确替换文件中的文本。

```
参数: filePath, oldString, newString
示例: 修改文件中的某段文字
```

### 4. bash - 执行命令

在终端执行命令行命令。

```
参数: command (命令)
示例: 执行 npm install
```

### 5. glob - 搜索文件

根据模式搜索文件。

```
参数: pattern (如 **/*.ts)
示例: 搜索所有 TypeScript 文件
```

### 6. grep - 搜索内容

在文件中搜索文本内容。

```
参数: pattern (搜索内容), filePattern (文件类型)
示例: 搜索包含 "function" 的文件
```

## 九、常见问题

### Q1: 提示 "未设置 MINIMAX_API_KEY"

需要创建 `.env` 文件并配置 API Key：

```bash
echo MINIMAX_API_KEY=你的密钥 > .env
```

### Q2: Windows 下命令执行失败

AI 模型会自动判断操作系统并生成对应的命令。如果遇到问题，可以在任务中说明操作系统。

### Q3: 工具执行多次

如果 AI 持续调用工具而不返回结果，可能是工具执行结果不理想。可以尝试：
1. 使用 `--max-iterations 5` 限制迭代次数
2. 在任务描述中更清晰地说明需求

### Q4: 对话历史太长

使用 `clear` 命令清除对话历史，或者使用单次任务模式。
